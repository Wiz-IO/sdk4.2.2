Overview

This section describes the GAP APIs for Bluetooth Low Energy only.

The Bluetooth Low Energy GAP profile defines the generic procedures related to discovery of Bluetooth enabled devices (idle mode procedures) and link management aspects of connecting to Bluetooth enabled devices (connecting mode procedures). It also defines procedures related to the use of different security levels and includes common format requirements for parameters accessible on the user interface level.

How to use this module.

  • Step1: Mandatory, implement the API ble_gap_common_callback() to handle the events sent by the Bluetooth Low Energy GAP.
  • Step2: Mandatory, set the Bluetooth Low Energy address after the Bluetooth powers on.
    • Sample code:
      int32_t main(void)
      {
      ble_address_t addr;
      //Do something, such as power on the Bluetooth enabled device.
      addr.address[0] = 0xC0;
      addr.address[1] = 2;
      addr.address[2] = 3;
      addr.address[3] = 4;
      addr.address[4] = 5;
      addr.address[5] = 0xC4;
      ble_gap_set_random_address(&addr);
      //Do something, such as create a connection.
      return 0;
      }

ble_status_t ble_gap_set_advertising_params ( uint16_t  min_interval,

ble_address_type_t  peer_addr_type,
const ble_address_t *  peer_addr,
uint8_t  channel_mask,
uint8_t  filter_policy 
)
Deprecated:
This function sets the advertising parameters.
Parameters
[in]min_intervalis the minimum advertising interval for undirected / directed advertising. Range: 0x0020 to 0x4000. Default: N = 0x0800 (1.28 second). Time = N * 0.625 msec.
[in]max_intervalis the maximum advertising interval for undirected / directed advertising. Range: 0x0020 to 0x4000. Default: N = 0x0800 (1.28 second). Time = N * 0.625 msec.
[in]advertising_typeis the type of advertising.
[in]own_addr_typeis the address used by the advertiser.
[in]peer_addr_typeis the type of peer_addr for directed advertising.
[in]peer_addris the address of device advertising used in the directed advertising.
[in]channel_maskis the channel mask used for directed advertising. xxxxxxx1b: Use channel 37. xxxxxx1xb: Use channel 38. xxxxx1xxb: Use channel 39. xxxxx111b: Default (all channels enabled). 00000000b: Invalid.
[in]filter_policy0x00: Default, process scan and connection requests from all devices. 0x01: Process connection requests from all devices and only scan requests from devices that are in the white list. 0x02: Process scan requests from all devices and only connection requests from devices that are in the white list. 0x03: Process scan and connection requests only from devices in the white list.
Returns
BLE_STATUS_SUCCESS, if the operation completed successfully, otherwise an error occurred.
Note
Advertiser must be stopped before setting up the parameters.
Example

2 {
3  ble_gap_stop_advertiser();
4  ble_gap_set_advertising_params(
5  0x100,
6  0x100,
7  BLE_GAP_ADVERTISING_CONNECTABLE_UNDIRECTED,
8  BLE_ADDRESS_TYPE_RANDOM,
9  BLE_ADDRESS_TYPE_PUBLIC,
10  NULL,
11  0x7,

13  );
14  return;
15 }
